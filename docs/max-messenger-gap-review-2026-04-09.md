# ZavodGram — проверка по списку замечаний к MAX (code audit)

Дата аудита: **09.04.2026**.

Ниже — разбор по коду: что уже реализовано в ZavodGram, что реализовано частично, и чего нет.

## 1) Что уже есть (совпадает с запросами пользователей)

1. **Цитирование / ответы на сообщение** — есть `replyToId` в модели и обработка в API/WS/UI.
2. **Реакции на сообщения** — есть модель `MessageReaction`, REST и WebSocket обновления.
3. **Комментарии под постами канала + отключение комментариев** — есть `commentsEnabled`, модерация комментариев.
4. **Публичная ссылка канала и вход по ссылке** — есть `channelSlug`, `GET /chats/public/:slug`, `POST /join`.
5. **Голосовые сообщения + расшифровка** — запись в web, загрузка аудио, endpoint транскрибации.
6. **Несколько сессий (multi-device)** — есть модель `Session`.
7. **Поиск по сообщениям** — есть endpoint `/messages/search`.
8. **Сквозной тип чата (SECRET) и флаг encrypted** — есть в схеме и UI-индикации.

## 2) Частично есть (нужна доработка)

1. **“Сквозное шифрование”**: в коде есть тип `SECRET`, поле `publicKey` и флаг `encrypted`,
   но нет полноценного протокола E2E (обмена ключами/криптографии на клиенте/проверки целостности).
2. **Приватность медиа (обновлено)**: доступ к медиа защищён проверкой membership; в web-клиенте
   переведено на загрузку через `Authorization: Bearer` и blob URL вместо `?token=` в ссылках.
3. **Копирование части текста (обновлено)**: добавлен пункт “Копировать фрагмент” в context-menu
   сообщения (берётся выделенный текст, если он есть).
4. **Каналы и управление**: базовое управление есть (баны, роли, муты комментариев),
   но нет удобного слоя “для авторов” (каталог, монетизация, аналитика по доходу).

## 3) Чего сейчас нет (функциональные гэпы)

1. **Закрепы (pinned messages)** — в схеме/роутах/клиенте не найдено.
2. **Темы в группах (topics/threads как отдельные ветки)** — нет сущности и API.
3. **Опросы/голосовалки** — нет модели/endpoint/UI.
4. **Архив чатов** — нет статуса archived в `ChatMember` и UI-режима архива.
5. **Истории (stories)** — нет backend/web реализации.
6. **Перевод текста / части текста** — нет endpoint/UI.
7. **Несколько аккаунтов в одном клиенте (быстрый switch account)** — нет в web-клиенте.
8. **Звонки/групповые звонки/видеозвонки** — в проекте нет call/WebRTC подсистемы.
9. **Публичный bot-platform API** — есть внутренний REST для клиента, но нет bot API/SDK/вебхуков.
10. **Платные подписки/донаты/рефералка/creator monetization** — нет платежной и партнерской подсистем.
11. **Редактор тем, кастомные рингтоны, расширенные аудио/видео-настройки звонков** — нет.
12. **Верифицированные аккаунты/боты (признак trusted)** — нет поля/процесса верификации в модели.

## 4) Выявленные риски/баги в текущем коде

1. **Потенциально некорректный `tempId` в WS-ack отправки**: сервер возвращает
   `tempId: data.chatId`, а не реальный временный id сообщения клиента.
2. **Mojibake в тексте ошибок media-модуля** (битые русские строки), что ухудшает UX диагностики.
3. **Регистрация только по `+7`** — технически уже ограничено regex-валидацией;
   это соответствует “запрету нероссийских SIM”, но может быть бизнес-риск для расширения аудитории.

## 5) Приоритет внедрения (что добавить сначала)

### P0 (срочно)
1. Дожать hardening выдачи медиа (короткоживущие signed URLs/ротация, аудит логов и рефереров).
2. Исправить WS-ack `tempId` для стабильного статуса отправки на клиенте.
3. Добавить тесты на доставку/редактирование/удаление/реакции в одном сценарии чата.

### P1 (паритет с Telegram/ожиданиями пользователей)
1. Закрепы.
2. Опросы.
3. Архив чатов.
4. Темы (topics) в группах.
5. Перевод текста/фрагмента.
6. Мультиаккаунт в клиенте.

### P2 (рост и удержание)
1. Stories.
2. Creator economy: подписки/донаты/рефералка.
3. Публичный bot API + webhooks.
4. Полноценное E2E (не только флаги в БД).

## 6) Короткий вывод “что есть / что добавить”

- **Есть:** ответы/цитирование, реакции, комментарии каналов, каналы по slug, голосовые + транскрипция, multi-session.
- **Добавить:** закрепы, опросы, архив, темы, звонки, перевод, stories, bot API, монетизацию авторов, полноценное E2E.

## 7) Дополнение по новым пунктам (скрины 6–8)

### 7.1 Функции, которых в текущем коде нет

1. **Закреплять больше 5 чатов / advanced pinning в списке** — механики pin в `ChatMember` нет.
2. **Быстрый поиск сообщений конкретного пользователя по клику на аватар** — отдельного фильтра `fromId` в UI/API нет.
3. **Переслать в чат, который ещё не открыт/не создан в списке** — в UI пересылка идёт по уже доступным чатам.
4. **Встроенный редактор изображений перед отправкой** — не реализован.
5. **Тесты/викторины как платформенный функционал** — не реализованы модели, endpoint и UI.
6. **Скрытие содержимого уведомлений при включённых уведомлениях** — отдельной privacy-настройки нет.
7. **Копирование изображения из буфера (paste image) в web** — обработчика paste/upload-from-clipboard нет.
8. **Принудительное сохранение изображений в JPEG** — отсутствует отдельная конвертация для пользовательской отправки.
9. **Верификация каналов/организаций внутри продукта** — нет поля/процесса верификации.
10. **Юридически значимая доставка/подтверждение вручения** — нет соответствующей модели статусов/квитанций.
11. **Блок “на 8 часов”, тонкие правила по входящим звонкам, полноценные звонки** — call-подсистема отсутствует.
12. **Desktop updater/теневые обновления** — отдельного desktop-клиента в этом репозитории нет (только web на Vite).

### 7.2 Частично покрыто

1. **“При клике на цитирование перейти к исходному сообщению”** — в web уже есть скролл к reply target,
   но не реализована явная “кнопка назад к месту чтения”.
2. **“Сохранение переписки между устройствами”** — данные хранятся серверно и есть multi-session,
   но проблемы вида “слетел аккаунт/всё потерялось” могут быть в потоке регистрации/восстановления, это требует runtime-инцидент-анализа.
3. **“Копировать часть сообщения”** — в web обычно можно выделять текст браузером,
   но отдельного UX-инструмента и контекстной команды “копировать фрагмент” нет.

### 7.3 Что нельзя подтвердить только статическим чтением кода

1. Самопроизвольные закрытия приложения при внешних share-операциях.
2. Реальная стабильность аккаунтов “слетает третий раз”.
3. Некорректная работа push/звука уведомлений на устройствах пользователей.
4. Ограничения старых версий Android/iOS (в этом репо нет мобильных клиентов).

### 7.4 Как обновить roadmap с учётом новых пунктов

- **P0:**
  1) incident-review по “слёту аккаунтов” и потере истории;
  2) аудит уведомлений (звук/прочитано/privacy contents);
  3) исправление UX reply-navigation (вперёд/назад).
- **P1:**
  1) pin chat;
  2) фильтр сообщений по автору;
  3) пересылка в любой чат/контакт;
  4) paste image в web;
  5) встроенный image editor light.
- **P2:**
  1) channel/org verification;
  2) юридически значимые квитанции доставки;
  3) образовательный модуль тестов/опросов;
  4) полноценная calls-платформа.
